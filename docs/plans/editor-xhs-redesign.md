# 小红书编辑器区域整体重构（规格）

> 编辑器区域需要整体重构。当前的编辑器太像一个普通文本框，和小红书草稿箱没有差异化。需要让用户在编辑过程中时刻感受到AI辅助的价值。以下是完整的修改计划：

---

## 1. 封面区域重构

删掉封面类型pill选择器（实拍图/文字封面/拼图/对比图/清单图全部删除）。删掉多图上传逻辑。改为以下结构：

**上方：封面预览区（3:4比例）**

- 没有上传照片时：显示Skill返回的封面建议卡片（推荐类型 + 画面描述 + 封面大字），背景为浅灰虚线框，引导用户"上传你拍的照片"
- 用户上传照片后：用原图 blob URL 作预览（不做客户端裁切叠字）；成图与画面内大字仅经 ```json:cover``` 生图 API
- 封面大字默认用 Skill 建议的文字，用户在预览图下方输入框编辑后随确认生成发给助手
- 预览下方两个按钮：「更换照片」+「下载封面」（导出当前预览/成图，不经本地 canvas 合字）
- 只支持上传1张照片作为封面，不做多图管理（多图是小红书APP的事）

**实现说明：** 上传预览为 `URL.createObjectURL` 原图；持久化见 `persistableImageUrl`（`src/lib/imageCompress.ts`）。

---

## 2. 正文编辑区增加AI辅助标记

不要用普通的textarea，改为按段落渲染的编辑区域：

- 正文按换行符分段，每段是一个独立的可编辑区域
- 每段左侧有一条3px的竖线颜色标记：
  - 绿色：用户修改过的段落（和originalDraft对比有差异）
  - 淡灰色：未修改的段落（仍然是AI原文）
- 竖线颜色实时更新

实现对比逻辑：见 `XhsPostEditor` 内按 `\n` 分段与 `originalParagraphs` 对比。

---

## 3. 底部状态栏改为三栏布局

一行三等分，紧凑排列：

**左栏：** 字数统计（实时）

**中栏：** 原创度 — 横向进度条 + 百分比数字 + 合规色标（>=60% 绿，40–59% 黄，<40% 红），debounce 1 秒

**右栏：** 风格学习 X/10 + 微型进度条，满 10 条时变绿并提示

---

## 4. 新增：AI建议侧边面板

- 默认折叠：灯泡图标 + badge（建议条数）
- 展开：质检阶段建议列表；每条：位置（第X段）+ 内容 +「采纳」
- 采纳：写入对应段落并标记已采纳
- 阶段二无数据：提示「编辑完成后提交质检，获取AI建议」

---

## 5. 质检触发方式

- 去掉固定「提交质检」大按钮
- 用户停止编辑 5 秒后，中栏原创度旁出现「深度质检」文字链接
- 点击 WebSocket 发送「继续」；未连接时前端轻量评分展示

---

## 改完后的编辑器结构（从上到下）

```
┌─────────────────────────────────┐
│  封面预览（3:4）                  │
│  [更换照片] [下载封面]            │
├─────────────────────────────────┤
│  标题输入（大字无边框）      x/20 │
├─────────────────────────────────┤
│ ▎ 段落1（灰色=AI原文）          │    [💡3] AI建议面板
│ ▊ 段落2（绿色=用户改过）        │
├─────────────────────────────────┤
│  #标签  [+]                      │
├─────────────────────────────────┤
│  字数 │ 原创度 ████ 67% │ 风格 │
└─────────────────────────────────┘
```
