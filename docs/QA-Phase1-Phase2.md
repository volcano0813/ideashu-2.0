# QA 验收清单（Phase 1 / Phase 2）

## 前置
1. 启动项目：`npm run dev`
2. 用浏览器打开：
   - Phase 1 创作工作台：`/`
   - Phase 1 素材银行：`/material-bank`
   - Phase 2 热点看板：`/hot-board`
   - Phase 2 知识库完整版：`/knowledge-base`
3. 本地存储键（用于排错）：
   - `ideashu.materials.v1`
   - `ideashu.posts.v1`
   - `ideashu.draftSession.v1`
   - `ideashu.pendingDraft.v1`

## Phase 1：创作链路联动
1. 素材银行 CRUD
   - 打开 `/material-bank`
   - 点击“添加素材”，填写 `类型 + 内容 + 主题标签(逗号分隔)`，保存后列表应立刻出现
   - 点击任意卡片右侧“删除”，确认后列表应移除
2. 素材 -> 工作台自动进入 stage2
   - 在素材银行点击“基于此素材创作”
   - 页面跳转到 `/` 后，应自动进入编辑器 `Stage 2`（不再需要手动在 stage1 里选题）
   - 左侧对话区应显示“提交质检”按钮
3. stage1 -> stage4（模拟对话驱动）
   - 返回 `/` 后按流程操作：
     1. 在 stage1 选择一个方向按钮
     2. 编辑器应自动加载草稿并允许你修改 2-3 处（修改次数/风格学习进度应增加）
     3. 点击左侧“提交质检”后，编辑器应进入 stage3 并展开质量评分面板
     4. 点击左侧“确认定稿”后，编辑器应进入 stage4（正文/标题变为只读），显示“复制全文/保存”
4. 编辑器可见性与计算项
   - 被修改过的段落左侧应出现 3px 黄色竖线
   - 底部“原创度环形”应随修改次数变化（并可点击展开原创度报告）
   - 底部“风格学习进度条”应显示 `X/10`，当 X>=10 时进度条变绿并提示“可触发风格分析”

## Phase 1：保存到知识库
1. 在 stage3/stage4 任意点击“保存到知识库 / 保存”
2. 打开 `/knowledge-base`
   - 列表应出现刚才保存的条目
   - 点击“基于此创作”应跳回 `/` 并自动进入 stage2

## Phase 2：页面渲染与跳转
1. 热点看板
   - 打开 `/hot-board`
   - 使用筛选器（最低热度/生命周期/平台来源）应即时影响卡片列表
   - 点击“用这个写”应跳回 `/`，并自动载入草稿进入 stage2
2. 账号管理/风格学习/数据追踪/设置
   - 分别打开：
     - `/accounts`
     - `/style-learning`
     - `/data-tracking`
     - `/settings`
   - 页面应正常渲染，不应报错

## WS 适配层（只做静态/调用层检查）
1. 项目构建应通过（`npm run build`）
2. `src/lib/openclawClient.ts` 应可被 import（mock 模式保证不可用时不阻塞页面）
